### Overview
I completed an 8-week full-time software engineering internship at **SmugMug**, working on the **Growth team**. My work spanned both frontend and backend systems and involved contributing to live production code used by millions of users.

This was my first experience working inside a large, mature production codebase, and it gave me hands-on exposure to real engineering processes: sprint planning, code review, QA, testing, and deployment.

---

### Area of Work
- Frontend and backend software development
- Working within a large **monorepo**
- Contributing to **production features and infrastructure upgrades**
- Collaborating with designers, frontend engineers, backend engineers, and QA

---

### Engineering Process & Takeaways
Working at SmugMug exposed me to a highly structured and scalable engineering workflow:

- **Security & layered architecture**  
  Learned how large production systems are structured with multiple layers of validation, review, and protection.

- **Tickets & sprints (Jira)**  
  Used Jira for sprint planning, task allocation, and prioritization in a fast-moving production environment.

- **Git, QA, and PR process**  
  Learned the full lifecycle of a change:
  - Feature development
  - Pull request review
  - QA validation
  - Production deployment

- **Modern + legacy codebases**  
  Gained experience navigating and modifying both modern and legacy systems without breaking existing behavior.

- **Frontend + backend development**  
  Worked across the stack using technologies including **PHP, React, and Next.js**.

- **Sandboxed environments**  
  Used internal sandbox tooling to safely test and validate changes without impacting production systems.
